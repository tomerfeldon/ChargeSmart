# Deploying ChargeSmart to a live URL

Three tiers, three homes: **Supabase** (database, already live) · **Render** (FastAPI
backend) · **Vercel** (React frontend). The database is already set up and seeded, so
only the two app tiers need deploying.

Prerequisites: the repo is on GitHub (`tomerfeldon/ChargeSmart`) and Supabase is running.
Have your Supabase **Session pooler** connection string ready (the same one in
`backend/.env`).

---

## Part A — Backend on Render

1. Go to **https://render.com** → sign up / log in with GitHub.
2. **New +** → **Blueprint** → pick the `ChargeSmart` repo. Render reads `render.yaml`
   and proposes a service named **chargesmart-api**. (If Blueprint isn't offered: **New +
   → Web Service**, root directory `backend`, build `pip install -r requirements.txt`,
   start `uvicorn app.main:app --host 0.0.0.0 --port $PORT`.)
3. Before the first deploy, set the secret env vars (Environment tab):
   - `DATABASE_URL` = your Supabase Session pooler URI (with the password).
   - `CHARGESMART_CORS_ORIGINS` = leave blank for now (set in Part C).
   - `CHARGESMART_JWT_SECRET` is auto-generated by the blueprint.
4. Deploy. When it's live, copy the service URL, e.g. `https://chargesmart-api.onrender.com`.
5. Verify: open `<backend-url>/health` → `{"status":"ok"}`, and `<backend-url>/docs`.

> ⚠️ Render's free tier sleeps after ~15 min idle; the first request then takes ~50 s
> (cold start). Hit `/health` a minute before a demo to wake it.

---

## Part B — Frontend on Vercel

1. Go to **https://vercel.com** → sign up / log in with GitHub.
2. **Add New… → Project** → import the `ChargeSmart` repo.
3. Set **Root Directory** = `frontend` (Framework Preset auto-detects **Vite**).
4. Add an Environment Variable:
   - `VITE_API_BASE` = your Render backend URL from Part A (e.g.
     `https://chargesmart-api.onrender.com`).
5. **Deploy**. Copy the resulting URL, e.g. `https://charge-smart.vercel.app`.

---

## Part C — Connect them (CORS)

1. Back in **Render** → chargesmart-api → **Environment** → set
   `CHARGESMART_CORS_ORIGINS` = your Vercel URL (e.g. `https://charge-smart.vercel.app`).
   Save → Render redeploys.
2. Open the Vercel URL, log in as `manager@chargesmart.test` / `manager123`.
3. (Optional) Re-seed the demo scenario any time: locally run
   `py scripts/seed_demo_sessions.py` — it writes to the same Supabase the live site reads.

That's it — share the Vercel URL with your professor.

---

## Notes
- The backend reads `DATABASE_URL`, `CHARGESMART_JWT_SECRET`, `CHARGESMART_CORS_ORIGINS`,
  and optional `ANTHROPIC_API_KEY` from environment variables (no `.env` in production).
- All three deployed tiers talk to the **same Supabase database**, so data seeded locally
  appears on the live site.
- Costs: Supabase free tier, Render free web service, Vercel hobby — all $0 for this scale.
